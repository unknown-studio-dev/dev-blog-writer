# Viblo Formatting Guide

## Key features to use

**Markdown support:** Viblo uses full GitHub-flavored markdown. All standard elements work: headers, bold/italic, code blocks, tables, blockquotes, task lists, and footnotes.

**Mermaid diagrams:** Viblo renders ` ```mermaid ` code blocks natively. Use diagrams freely — they display well and the Vietnamese dev community appreciates visual explanations.

**Table of contents:** Viblo auto-generates TOC from headers. Use H2/H3 consistently for good navigation.

**Code blocks:** Excellent syntax highlighting. Specify the language. Viblo readers expect well-formatted, runnable code examples.

**Series:** Group related posts into a series. Good for build-in-public journeys or multi-part tutorials.

**Tags:** Up to 5 tags per post. Use both Vietnamese and English tags for discovery (e.g., `javascript`, `backend`, `kinh-nghiem`).

## Viblo-specific strengths

**Vietnamese tech community:** Viblo is THE platform for the Vietnamese dev community. Write naturally in Vietnamese with English tech terms — this is what readers expect.

**Technical depth:** Viblo readers are developers and engineers. Go deep. Show real code, real architecture, real numbers. Surface-level posts don't get traction.

**Clip feature:** Readers "clip" (bookmark) useful posts. Practical, reference-worthy content gets clipped and reshared.

**Organizations:** If the user posts under an organization, mention the org context. Company engineering blogs on Viblo get good visibility.

**SEO in Vietnamese:** Viblo posts rank well in Vietnamese Google searches. Use Vietnamese in headers and descriptions for discoverability.

## Writing for Viblo

Viblo has a distinctive voice in the Vietnamese dev community:

- **"Tản mạn" style is beloved** — reflective, personal posts that share real experience
- **Mix Vietnamese + English naturally** — "Mình đã dùng Redis để cache, giảm latency từ 200ms xuống 15ms"
- **Address readers as "anh em"** (casual) or "mọi người" (neutral)
- **Show vulnerability** — posts about failures and lessons learned perform well
- **Include actionable takeaways** — Viblo readers want to learn something they can apply

## What to avoid on Viblo

- Don't write purely in English (the community expects Vietnamese or mixed)
- Don't translate technical terms that are universally used in English (deploy, database, cache, API — keep these in English)
- Don't write listicles without depth ("10 thư viện JavaScript hay" without real context)
- Don't skip code examples for technical posts (Viblo readers expect hands-on content)
- Don't be overly formal — Viblo's culture is peer-to-peer, not academic

## Output structure for Viblo

```markdown
# Headline Here

> Mô tả ngắn / Short description for SEO and social preview

[Body with H2/H3 sections, code blocks with language hints, Mermaid diagrams where relevant]

---

**Headline options:**
1. Option A
2. Option B
3. Option C

**Suggested Viblo tags:** tag1, tag2, tag3, tag4, tag5
```
