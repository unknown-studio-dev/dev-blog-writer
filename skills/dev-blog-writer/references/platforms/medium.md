# Medium Formatting Guide

## Key features to use

**Subtitle/kicker:** Medium shows a subtitle preview in feeds. Keep it under 140 characters.

**Featured image:** Medium strongly favors posts with a cover image. It appears in feeds and social shares. Use a relevant screenshot, diagram, or custom graphic.

**Headers:** Use H2 for main sections. Medium's clean typography makes well-structured posts look great.

**Code blocks:** Use triple backticks with language hints. Medium renders them in a monospace font with a gray background. Keep snippets short since Medium's code rendering isn't great on mobile.

**Pull quotes:** Use blockquotes (>) for key takeaways. Medium styles these prominently with a green left border.

**Gists:** For longer code, embed GitHub Gists instead of inline code blocks. They render better and readers can star/fork them.

## Medium-specific strengths

**Publications:** Submit to relevant publications (Better Programming, Towards Data Science, The Startup) for much wider reach than self-publishing.

**SEO:** Medium has strong domain authority. Posts rank well in Google. Include relevant keywords naturally.

**Claps:** End with something that makes readers want to engage. The clap system rewards posts that provoke a reaction.

## What to avoid on Medium

- Don't use bullet points excessively (Medium's styling makes them look dense)
- Don't start with "In this article, I will..." (Medium readers scroll past this)
- Don't publish behind the paywall if you want maximum reach for dev content
- Don't forget to add 3-5 tags. They determine which feeds your post appears in
- Don't use H1 in the body (Medium reserves H1 for the title)

## Output structure for Medium

```markdown
# Headline Here

## Subtitle that provides context

![Cover image description](placeholder-for-cover-image)

[Body with H2 sections, pull quotes, embedded gists for long code]

---

**Alternative headlines:**
1. Option A
2. Option B
3. Option C

**Suggested Medium tags:** tag1, tag2, tag3, tag4, tag5
```
