# Hashnode Formatting Guide

## Key features to use

**Series:** Group related posts into a series. Readers can follow the series and get notified. Great for build-in-public updates.

**Cover image:** Hashnode displays cover images prominently. Include one for better feed placement.

**Table of contents:** Hashnode auto-generates a TOC from your headers. Use H2 and H3 consistently.

**Code blocks:** Excellent code rendering with syntax highlighting. Specify the language after triple backticks. Supports line highlighting with `{1,3-5}` notation.

**Embeds:** Supports GitHub gists, CodePen, CodeSandbox, YouTube, and Twitter embeds natively. Use these for interactive demos.

**Custom domain:** If the user has a custom blog domain on Hashnode, link to that instead of hashnode.dev.

## Hashnode-specific strengths

**Developer community:** Hashnode's audience is almost entirely developers. You can go deeper technically than on Medium or Substack without losing readers.

**SEO:** Posts on Hashnode (especially custom domains) get good Google indexing. Use relevant keywords in headers.

**Tags:** Up to 5 tags per post. Use specific tags (e.g., "nextjs" not just "javascript") for better discovery.

**Hashnode AI:** Readers may discover posts through Hashnode's AI-powered feed. Make the first paragraph count.

## What to avoid on Hashnode

- Don't write beginner tutorials unless genuinely adding new insight (the platform is saturated with them)
- Don't neglect the canonical URL setting if cross-posting from another blog
- Don't skip tags. They're the primary discovery mechanism

## Output structure for Hashnode

```markdown
# Headline Here

> Subtitle or key takeaway as a blockquote

![Cover image description](placeholder-for-cover-image)

[Body with H2/H3 sections, code blocks with language hints, embeds]

---

**Alternative headlines:**
1. Option A
2. Option B
3. Option C

**Suggested Hashnode tags:** tag1, tag2, tag3, tag4, tag5
```
